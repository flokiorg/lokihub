
This release focuses on application lifecycle improvements, backend observability, and a more robust user interface, including the introduction of a tray icon for background execution.

## Major Changes

### 🛠 Application Lifecycle & Tray Support
- **Tray Icon Support:** Introduced a system tray icon for Darwin (macOS), Linux, and Windows.
- **Background Execution:** Lokihub now continues to run in the background when the main window is closed. Users can fully exit the application via the tray menu.
- **Dock Management:** On macOS, the application icon is dynamically shown or hidden from the dock based on window visibility.

### 🔌 Backend Refactor & API Enhancements
- **Service-Owned Loggers:** Refactored the entire backend to use contextual, service-owned loggers. This improves observability by tagging logs with their respective components (e.g., `http`, `transactions`, `lsps`).
- **Invoice API Enhancements:** Updated the `MakeInvoice` API to support `appId` and `descriptionHash`, allowing for better integration and attribution of payments to specific applications.
- **Stability Improvements:** Added comprehensive test suites for LSPS management and Swap services.

### 🎨 Frontend UI/UX Improvements
- **New Help Section:** Added a dedicated Help and FAQ section to assist users with common tasks and troubleshooting.
- **Error Resilience:** Implemented a global `ErrorBoundary` to prevent the entire application from crashing due to localized UI failures.
- **Localization:** Expanded localization support with new help content for multiple languages.
- **Component Polish:** Refined various UI components including `AppSidebar`, `AppHeader`, and transaction lists for better consistency and responsiveness.

## Technical Notes
- Integrated `github.com/getlantern/systray` for cross-platform tray management.
- Implemented SVG-to-PNG rendering for tray icons to ensure high-quality visuals across different resolutions.
- Updated backend services to inject loggers, facilitating better testing and log filtering.
